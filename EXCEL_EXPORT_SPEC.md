# 財務集計エクセル出力機能 仕様書

## 概要

Notionデータベースから商品データを取得し、企業別・月別にピボット集計したエクセルファイルを出力する機能。

**重要**:
- 売上・利益の計算：売却済みデータのみ使用
- 仕入高の計算：在庫状況に関係なく全データ使用

---

## 機能仕様

### 1. データ取得

#### 取得対象データ（2種類）

##### 1-1. 売却済みデータ（売上・利益計算用）
- **データベース**: 商品一覧（ID: `1d254e6206d881bb9e88d2e7ffb90444`）
- **フィルタ条件**:
  - `在庫状況` = "売却済み"
  - `売却日` が指定期間内（12ヶ月分）
- **使用目的**: 売上金、販売利益の集計

##### 1-2. 全データ（仕入高計算用）
- **データベース**: 同上
- **フィルタ条件**:
  - `Created time` が指定期間内（12ヶ月分）
  - 在庫状況は**不問**（在庫中、販売中含む）
- **使用目的**: 仕入れ原価の集計

#### 取得期間
- 開始年月を選択可能（デフォルト: 2025年6月）
- 開始月から**12ヶ月分**のデータを自動取得
- ページネーション対応（全データを取得）

#### 取得データ項目
| 項目名 | 用途 | データソース |
|--------|------|-------------|
| 仕入れ先 | 業販の企業分類 | 売却済み / 全データ |
| 販売媒体 | 小売の企業分類 | 売却済みのみ |
| 売却日 | 月別集計の基準日（売上・利益） | 売却済みのみ |
| Created time | 月別集計の基準日（仕入高） | 全データ |
| 売上金 | 売上集計 | 売却済みのみ |
| 純利益 | 販売利益集計（手数料・送料込み） | 売却済みのみ |
| 仕入れ原価 | 仕入高集計 | 全データ |

---

## 2. ピボット集計

### 5つの集計セクション

#### セクション1: 企業別売上（業販）
- **集計軸**: 仕入先 × 売却年月
- **集計値**: 売上金（何も引かない）
- **データソース**: 売却済みデータのみ
- **意味**: 各仕入先から仕入れた商品を売却した売上金額

#### セクション2: 企業別販売利益（業販）
- **集計軸**: 仕入先 × 売却年月
- **集計値**: 純利益（手数料・送料含む全コスト控除後）
- **データソース**: 売却済みデータのみ
- **意味**: 各仕入先から仕入れた商品の販売利益

#### セクション3: 企業別売上（小売）
- **集計軸**: 販売媒体 × 売却年月
- **集計値**: 売上金（何も引かない）
- **データソース**: 売却済みデータのみ
- **意味**: 各販売媒体での売上金額

#### セクション4: 企業別販売利益（小売）
- **集計軸**: 販売媒体 × 売却年月
- **集計値**: 純利益（手数料・送料含む全コスト控除後）
- **データソース**: 売却済みデータのみ
- **意味**: 各販売媒体での販売利益

#### セクション5: 企業別仕入高
- **集計軸**: 仕入先 × 登録年月（Created time）
- **集計値**: 仕入れ原価
- **データソース**: **全データ**（在庫状況不問）
- **意味**: 各仕入先からの仕入金額（売れていなくても計上）
- **重要**: 売却日ではなく、登録日（Created time）を基準に集計

## 3. エクセル出力形式

### ピボットテーブル構造

**5つのセクションが縦に並ぶ形式**

```
企業別売上（業販）
                2025年6月  2025年7月  2025年8月  ...  2026年5月   計
企業A              1,000      2,000      1,500          3,000    50,000
企業B                500        800      1,200          1,000    20,000
合計               1,500      2,800      2,700          4,000    70,000

企業別販売利益（業販）
                2025年6月  2025年7月  2025年8月  ...  2026年5月   計
企業A                500        900        600          1,200    25,000
企業B                200        300        500            400    10,000
合計                 700      1,200      1,100          1,600    35,000

企業別売上（小売）
                2025年6月  2025年7月  2025年8月  ...  2026年5月   計
メルカリ1           1,000      1,500      1,200          2,000    40,000
Yahoo!               500        800      1,000            800    15,000
合計               1,500      2,300      2,200          2,800    55,000

企業別販売利益（小売）
                2025年6月  2025年7月  2025年8月  ...  2026年5月   計
メルカリ1             400        700        500            900    18,000
Yahoo!                150        250        400            300     6,000
合計                 550        950        900          1,200    24,000

企業別仕入高
                2025年6月  2025年7月  2025年8月  ...  2026年5月   計
企業A                800      1,500      1,200          2,500    45,000
企業B                600      1,000        800          1,500    25,000
合計               1,400      2,500      2,000          4,000    70,000
```

### 書式設定
- **セクションタイトル**: 太字、14pt
- **ヘッダー行**: 太字、中央揃え
- **数値**: カンマ区切り形式（`#,##0`）
- **列幅**: 企業名列 20、月列 12
- **合計行**: 太字
- **セクション間**: 1行の空行

### ファイル名
```
財務集計_{開始年}年{開始月}月-{終了年}年{終了月}月.xlsx
```
例: `財務集計_2025年6月-2026年5月.xlsx`

## 5. 使用方法

### 操作手順
1. アプリケーションを起動
   ```bash
   python local_server/run_proxy.py
   ```

2. 設定タブでNotion API KeyとDatabase IDを設定

3. Excel出力タブを開く

4. 財務集計期間選択で開始年月を選択
   - デフォルト: 2025年6月
   - 選択可能範囲: 2020年〜2030年、1月〜12月

5. 「財務集計データを取得」ボタンをクリック
   - 進捗バーが表示される
   - 取得完了後、件数と期間が表示される

6. 「財務集計Excelを保存」ボタンをクリック
   - ファイル保存ダイアログが表示される
   - 保存先を選択して保存

## 6. 技術仕様

### 使用ライブラリ
- **flet**: UI フレームワーク
- **pandas**: データ処理・ピボット集計
- **openpyxl**: Excel ファイル生成
- **notion-client**: Notion API クライアント

### データフロー
```
Notion API（2回のクエリ）
    ↓
1. 売却済みデータ取得（ページネーション）
   - 在庫状況 = "売却済み"
   - 売却日でフィルタ
    ↓
2. 全データ取得（ページネーション）
   - Created timeでフィルタ
   - 在庫状況不問
    ↓
parse_notion_results（DataFrame変換）
    ↓
create_pivot_sections（ピボット集計）
   - pivot_df: 売却済みデータ
   - purchase_df: 全データ
    ↓
export_pivot_excel（Excel出力）
   - 5セクション構成
```

### ページネーション処理
- Notion APIのデフォルト制限: 100件/リクエスト
- `has_more`と`next_cursor`を使用して全データを取得
- 売却済みと全データそれぞれで個別に取得
- 全結果を結合してDataFrameに変換

## 7. 注意事項

### データの正確性
- **売上・利益**: 売却日が未設定の場合、そのレコードは集計に含まれない
- **仕入高**: Created timeを基準に集計、売却日は不要
- 純利益はNotionで計算済みの値を使用
- 仕入れ原価が0の場合も0として集計される

### 在庫状況の扱い
- **売上（業販・小売）**: 売却済みのみ
- **販売利益（業販・小売）**: 売却済みのみ
- **仕入高**: **全データ**（在庫中、販売中を含む）

### パフォーマンス
- データ量が多い場合、取得に時間がかかる場合がある（2回のAPI呼び出し）
- プログレスバーで進捗を確認可能

### エラーハンドリング
- API接続エラー: "❌ エラー: {エラー内容}" を表示
- データなし: "⚠️ 該当期間のデータがありません" を表示

## 8. カスタマイズ

### 集計期間の変更

2. 返り値の辞書に追加
3. `export_pivot_excel`で自動的に出力される

### データ項目の追加
1. `parse_notion_results`メソッドで新しいプロパティを処理
2. `create_pivot_sections`で新しい列を作成
3. ピボットテーブルの`values`に追加

## 9. トラブルシューティング

### データが取得できない
- Notion API KeyとDatabase IDが正しく設定されているか確認
- 在庫状況が「売却済み」になっているか確認
- 売却日が指定期間内にあるか確認

### 集計結果がおかしい
- 仕入先・販売媒体のURL除去が正しく動作しているか確認
- 数値フィールドが正しく変換されているか確認
- NotionのプロパティタイプがStatus型、Date型になっているか確認

### Excel保存エラー
- 保存先のパスに書き込み権限があるか確認
- ファイル名に使用できない文字が含まれていないか確認
- openpyxlライブラリがインストールされているか確認
