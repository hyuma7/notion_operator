# 財務集計エクセル出力機能 仕様書

## 概要

Notionデータベースから商品データを取得し、企業別・月別にピボット集計したエクセルファイルを出力する機能。

**重要**:
- **売上・利益**: 販売媒体のみを集計（メルカリ、Yahooフリマなど）
- **仕入高**: 仕入れ先のみを集計（RE、REFなど）
- カテゴリはNotionから取得した値をそのまま使用（制限なし）

---

## 機能仕様

### 1. データ取得

#### 取得対象データ（2種類）

##### 1-1. 売却済みデータ（売上・利益計算用）
- **データベース**: 商品一覧（ID: `1d254e6206d881bb9e88d2e7ffb90444`）
- **フィルタ条件**:
  - `在庫状況` = "売却済み"
  - `売却日` が指定期間内（12ヶ月分）
- **使用目的**: 売上金、販売利益の集計

##### 1-2. 全データ（仕入高計算用）
- **データベース**: 同上
- **フィルタ条件**:
  - `Created time` が指定期間内（12ヶ月分）
  - 在庫状況は**不問**（在庫中、販売中含む）
- **使用目的**: 仕入れ原価の集計

#### 取得期間
- 開始年月を選択可能（デフォルト: 2025年6月）
- 開始月から**12ヶ月分**のデータを自動取得
- ページネーション対応（全データを取得）

#### 取得データ項目

| 項目名 | タイプ | データソース | 説明 |
|--------|--------|-------------|------|
| 仕入れ先名 | ロールアップ（formula） | 全データ | 仕入れ先リレーションから取得 |
| 販売媒体名 | ロールアップ（formula） | 売却済みのみ | 販売媒体リレーションから取得 |
| 仕入先カテゴリ | ロールアップ（select） | 全データ | 仕入れ先のカテゴリ |
| 販売先カテゴリ | ロールアップ（select） | 売却済みのみ | 販売媒体のカテゴリ |
| 作業担当 | people | 全データ | 担当者名 |
| 売却日 | date | 売却済みのみ | 月別集計の基準日（売上・利益） |
| Created time | datetime | 全データ | 月別集計の基準日（仕入高） |
| 売上金 | number | 売却済みのみ | 売上集計 |
| 純利益 | formula | 売却済みのみ | 販売利益集計（手数料・送料込み） |
| 仕入れ原価 | formula | 全データ | 仕入高集計 |

## 2. データ処理

### ロールアップ処理
Notionのロールアッププロパティは以下の構造で返されます：

```json
{
  "type": "array",
  "array": [
    {
      "type": "formula",
      "formula": {
        "type": "string",
        "string": "メルカリ1"
      }
    }
  ]
}
```

システムは以下のロールアップタイプに対応：
- `formula` (string/number)
- `select`
- `title`
- `rich_text`
- `number`

### カテゴリ処理
- Notionから取得した値をそのまま使用（例：「市場」「ネット」「業販」「小売」など）
- 空の場合のみ「その他」に設定
- 動的にカテゴリを収集し、ソート順で表示

### データクリーニング

#### Notion URL除去
```
変換前: RE (https://www.notion.so/RE-27e54e6206d8806083e7e48503f5f080?pvs=21)
変換後: RE
```

#### 通貨形式変換
```
変換前: ￥1,000
変換後: 1000 (数値)
```

#### 空欄・未設定値の処理
- 企業名が空の場合: "不明"
- 数値が空の場合: 0
- カテゴリが空の場合: "その他"

## 3. ピボット集計

### 3つの集計セクション

#### セクション1: 企業別売上
- **集計軸**: 販売媒体 × 売却年月
- **集計値**: 売上金
- **データソース**: 売却済みデータのみ
- **意味**: 各販売媒体（メルカリ、Yahooフリマなど）での売上金額

#### セクション2: 企業別販売利益
- **集計軸**: 販売媒体 × 売却年月
- **集計値**: 純利益（手数料・送料含む全コスト控除後）
- **データソース**: 売却済みデータのみ
- **意味**: 各販売媒体での販売利益

#### セクション3: 企業別仕入高
- **集計軸**: 仕入先 × 登録年月（Created time）
- **集計値**: 仕入れ原価
- **データソース**: **全データ**（在庫状況不問）
- **意味**: 各仕入先（RE、REFなど）からの仕入金額
- **重要**: 売却日ではなく、登録日（Created time）を基準に集計

## 4. エクセル出力形式

### ピボットテーブル構造

**3つのセクションが縦に並ぶ形式**

```
企業別売上
                2025年6月  2025年7月  2025年8月  ...  2026年5月   計
メルカリ1          1,000      2,000      1,500          3,000    50,000
Yahooフリマ2         500        800      1,200          1,000    20,000
齊藤光計           1,500      2,800      2,700          4,000    70,000
ゆかり 齊藤計         0          0          0              0         0
ネット計           1,000      2,000      2,000          3,000    50,000
業販計               500        800        700          1,000    20,000
合計               1,500      2,800      2,700          4,000    70,000

企業別販売利益
                2025年6月  2025年7月  2025年8月  ...  2026年5月   計
メルカリ1            500        900        600          1,200    25,000
Yahooフリマ2         200        300        500            400    10,000
齊藤光計             700      1,200      1,100          1,600    35,000
ゆかり 齊藤計         0          0          0              0         0
ネット計             500        900        800          1,200    25,000
業販計               200        300        300            400    10,000
合計                 700      1,200      1,100          1,600    35,000

企業別仕入高
                2025年6月  2025年7月  2025年8月  ...  2026年5月   計
REF                  800      1,500      1,200          2,500    45,000
RE                   600      1,000        800          1,500    25,000
齊藤光計           1,200      2,200      1,800          3,500    60,000
ゆかり 齊藤計       200        300        200            500    10,000
ネット計           1,000      1,800      1,500          3,000    55,000
その他計             400        700        500          1,000    15,000
合計               1,400      2,500      2,000          4,000    70,000
```

### 行の構成

各セクションは以下の順序で行を出力：

1. **企業明細行**: 各企業（販売媒体または仕入先）の月別データ
2. **ユーザー別小計行**: 担当者ごとの小計（`{担当者名}計`）
   - 淡い背景色で区別
   - 太字フォント
3. **カテゴリ別小計行**: カテゴリごとの小計（`{カテゴリ名}計`）
   - 例：「ネット計」「業販計」「市場計」など
   - 薄青色の背景色
   - 太字フォント
4. **総計行**: 全体の合計（`合計`）
   - 薄橙色の背景色
   - 太字フォント

### 書式設定

- **セクションタイトル**: 太字、14pt
- **ヘッダー行**: グレー背景、太字、中央揃え
- **数値**: カンマ区切り形式（`#,##0`）
- **列幅**: 企業名列 25、月列 12
- **ユーザー別小計**: 担当者ごとに異なる淡い背景色
- **カテゴリ別小計**: 薄青色背景（#E0F2F7）
- **総計行**: 薄橙色背景（#FFE4B5）
- **セクション間**: 1行の空行

### ファイル名
```
財務集計_{開始年}年{開始月}月-{終了年}年{終了月}月.xlsx
```
例: `財務集計_2025年6月-2026年5月.xlsx`

## 5. 使用方法

### 操作手順
1. アプリケーションを起動
   ```bash
   python local_server/run_proxy.py
   ```

2. 設定タブでNotion API KeyとDatabase IDを設定

3. Excel出力タブを開く

4. 財務集計期間選択で開始年月を選択
   - デフォルト: 2025年6月
   - 選択可能範囲: 2020年〜2030年、1月〜12月

5. 「財務集計データを取得」ボタンをクリック
   - 進捗バーが表示される
   - 取得完了後、件数と期間が表示される

6. 「財務集計Excelを保存」ボタンをクリック
   - ファイル保存ダイアログが表示される
   - 保存先を選択して保存

## 6. 技術仕様

### 使用ライブラリ
- **flet**: UI フレームワーク
- **pandas**: データ処理・ピボット集計
- **openpyxl**: Excel ファイル生成
- **notion-client**: Notion API クライアント
- **pydantic**: データバリデーション

### アーキテクチャ

```
ui/export/
├── __init__.py          # モジュールエクスポート
├── ui.py                # UIコンポーネント（ExportTab）
├── service.py           # ビジネスロジック（ExportService）
└── schemas.py           # データスキーマ（Pydantic）
```

### データフロー
```
Notion API（2回のクエリ）
    ↓
1. 売却済みデータ取得（ページネーション）
   - 在庫状況 = "売却済み"
   - 売却日でフィルタ
    ↓
2. 全データ取得（ページネーション）
   - Created timeでフィルタ
   - 在庫状況不問
    ↓
_flatten_notion_page（ロールアップ・formula処理）
    ↓
Pydanticバリデーション（SoldRecord / PurchaseRecord）
    ↓
process_pivot_data（ピボット集計）
   - pivot_sales: 販売媒体の売上
   - pivot_profit: 販売媒体の利益
   - pivot_purchase: 仕入先の仕入高
   - カテゴリマッピング（動的収集）
   - 担当者マッピング（セクション別）
    ↓
generate_excel（Excel出力）
   - 3セクション構成
   - ユーザー別小計
   - カテゴリ別小計
   - 総計
```

### ページネーション処理
- Notion APIのデフォルト制限: 100件/リクエスト
- `has_more`と`next_cursor`を使用して全データを取得
- 売却済みと全データそれぞれで個別に取得
- 全結果を結合してPydanticモデルに変換

## 7. 注意事項

### データの正確性
- **売上・利益**: 売却日が未設定の場合、そのレコードは集計に含まれない
- **仕入高**: Created timeを基準に集計、売却日は不要
- 純利益はNotionで計算済みの値を使用
- 仕入れ原価が0の場合も0として集計される

### 集計対象の違い
- **企業別売上**: 販売媒体（メルカリ、Yahooフリマなど）のみ
- **企業別販売利益**: 販売媒体のみ
- **企業別仕入高**: 仕入先（RE、REFなど）のみ

### カテゴリの柔軟性
- Notionのカテゴリ値を制限なく使用可能
- 「市場」「ネット」「業販」「小売」「その他」など、任意の値に対応
- カテゴリは自動的にソート順で表示

### 担当者マッピング
- 売上・利益: 販売媒体と担当者の関連付け
- 仕入高: 仕入先と担当者の関連付け
- 各セクションで独立した担当者マッピングを使用

### パフォーマンス
- データ量が多い場合、取得に時間がかかる（2回のAPI呼び出し）
- プログレスバーで進捗を確認可能

### エラーハンドリング
- API接続エラー: "❌ エラー: {エラー内容}" を表示
- データなし: "⚠️ 該当期間のデータがありません" を表示
- ロールアップ処理エラー: デバッグ情報をコンソールに出力

## 8. デバッグ機能

### コンソール出力
以下のデバッグ情報がコンソールに出力されます：

```
DEBUG: 仕入れ先名 rollup_type=array, rollup={...}
DEBUG: 仕入れ先名 extracted values=['REF'], result=REF
DEBUG: 販売媒体名 rollup_type=array, rollup={...}
DEBUG: 販売媒体名 extracted values=['メルカリ1'], result=メルカリ1
DEBUG: 仕入先カテゴリ rollup_type=array, rollup={...}
DEBUG: 仕入先カテゴリ extracted values=['ネット'], result=ネット
DEBUG: 販売先カテゴリ rollup_type=array, rollup={...}
DEBUG: 販売先カテゴリ extracted values=['業販'], result=業販
DEBUG: 作業担当 people=[...], names=['齊藤光'], result=齊藤光
```

### デバッグ情報の読み方
- `rollup_type`: ロールアップの返却タイプ（array/number/date）
- `rollup`: 完全なロールアップオブジェクト
- `extracted values`: 抽出された値のリスト
- `result`: 最終的にセットされる値

## 9. トラブルシューティング

### データが取得できない
- Notion API KeyとDatabase IDが正しく設定されているか確認
- 在庫状況が「売却済み」になっているか確認（売上・利益の場合）
- 売却日が指定期間内にあるか確認

### 企業名が取得できない
- ロールアッププロパティが正しく設定されているか確認
- リレーション先のページにformulaプロパティがあるか確認
- デバッグ情報で `extracted values=[]` となっている場合、ロールアップ構造を確認

### カテゴリが「その他」になる
- Notionのselectプロパティに値が設定されているか確認
- ロールアップが正しくselectタイプを返しているか確認

### 担当者が表示されない
- peopleプロパティにユーザーが割り当てられているか確認
- ユーザーの`name`フィールドが存在するか確認

### 集計結果がおかしい
- 販売媒体と仕入先が正しく分離されているか確認
- カテゴリマッピングが正しく動作しているか確認
- NotionのプロパティタイプがStatus型、Date型になっているか確認

### Excel保存エラー
- 保存先のパスに書き込み権限があるか確認
- ファイル名に使用できない文字が含まれていないか確認
- openpyxlライブラリがインストールされているか確認
